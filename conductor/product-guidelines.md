# Product Guidelines — Austroads Data Info Hub

Technical and design standards for the Austroads Data Info Hub, optimized for Power Pages and AI-integrated development.

## 1. Core Architectural Mandates

### Power Pages Environment Rules
- **JS-Driven UI Components**: UI elements requiring `position:fixed` (like the AI Chat Drawer) must be built in JavaScript and appended to `document.body` to escape the Power Pages CSS transform wrapper.
- **Dual-Copy Pattern**: Any change to page content (HTML, CSS, or JS) must be synced to both the base `web-pages` path and the `content-pages` (en-US) sub-path in the deployment directory.
- **CSS Inlining**: Shared styles must be inlined in the `Austroads-Layout` web template `<style>` block. Browser caching of `.css` web files is unreliable in this environment.
- **Liquid Resilience**: JavaScript reading Liquid-rendered values (e.g., user data) must check for the presence of `{{` or `}}` to handle cases where Liquid fails to render.

### Event Handling
- **Surviving Re-renders**: Use **Inline `onclick`** for primary triggers (e.g., opening the AI drawer) and **Document-level delegation** for internal component events. This ensures handlers survive Power Pages DOM re-renders.
- **Conflict Prevention**: Never use both inline `onclick` and document delegation on the same element to avoid double-firing.

## 2. Design System (Style Guide 2021)

### Primary Palette
- **Pitch (Navy)**: `#3C3533` (Pantone Black 7)
- **Gold (Accent)**: `#F36F21` (Pantone 166)
- **Web Colour (Links)**: `#CC4C00` (Pantone 718)

### Domain Branding
- **NEVDIS**: `#CC4C00` (Orange) — Use `.nevdis-section` class.
- **TCA**: `#2789AF` (Teal) — Use `.tca-section` class.
- **General**: `#3C3533` (Charcoal) — Use for cross-domain content.

### Typography
- **Headings**: `Roboto Slab` (Google Fonts).
- **Body**: `Arial, Helvetica, sans-serif`.

## 3. AI Agent Standards (v2)

### Interaction Model
- **Router + Specialists**: Questions are first classified by a Router AI into NEVDIS, TCA, or GENERAL domains. Responses are then generated by specialist prompts using domain-specific Dataverse knowledge articles.
- **Rich Responses**: Bot answers must use Markdown (headers, lists, bold, code) which is rendered to HTML client-side.
- **Source Citations**: Mentions of data products should be automatically converted to links to the relevant detail pages using the internal `PRODUCT_MAP`.

### Technical Contract
- **History**: Always send the last 8 messages (4 exchanges) for context.
- **Timeout**: Handle the 15-20s AI Builder latency with a typing indicator and "Still thinking..." hints after 12s.
