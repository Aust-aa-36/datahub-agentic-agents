Plate-to-VIN (P2V)

A secure API service that converts an Australian registration plate and state into a VIN or chassis number — enabling authorised commercial brokers to verify vehicle identity in real time.

What it does

Plate-to-VIN (P2V) is a secure service that delivers vehicle data from NEVDIS to authorised commercial users (brokers). It converts a number plate and state into a VIN or chassis number, providing the current state of data only — no historical information is returned.

Who uses it

Access is restricted to five currently authorised brokers: VEHICLEID, EVOLVE, BLUEFLAG, INFOAGENT, and CARSALES. These brokers supply the data to their own downstream customers across fleet management, insurance, and automotive industries.

⚠️ NEVDIS is not currently processing new broker or P2V customer applications due to resource constraints. Existing resources are focused on supporting jurisdiction functions.

Use cases

Fleet vehicle identification and management, insurance verification prior to quoting, vehicle identity confirmation during purchase, and cross-referencing plate records against VIN-based systems.

Inputs

Input FieldTypeRequiredDescription
Plate NumberStringYesThe vehicle's registration plate number
State / TerritoryStringYesThe jurisdiction in which the plate is registered (e.g. NSW, VIC)
Enquiry MessageStringYesBroker-generated reference for the enquiry

Outputs

Output FieldTypeDescription
VINString17-character Vehicle Identification Number (1989+ vehicles)
Chassis NumberStringChassis number for pre-1989 vehicles without a VIN

Process

  1. Broker signs a NEVDIS P2V Data Access Agreement with Austroads.
  2. Broker system submits plate number + state via the API (JSON or XML format).
  3. NEVDIS queries the registration database for a matching current record.
  4. VIN or chassis number returned in real time to the broker system.

Terms of use

  • Access restricted to brokers holding a current NEVDIS P2V Data Access Agreement
  • Data provided 'as is' — current state only, no historical records returned
  • No liability for errors, omissions, or reliance on returned data
  • Use restricted to the purposes outlined in the Data Access Agreement
  • Downstream redistribution governed by individual broker agreements
  • NEVDIS is not currently accepting new broker applications
